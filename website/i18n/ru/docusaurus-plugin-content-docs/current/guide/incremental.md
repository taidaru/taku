# Инкрементальные таски

`unchanged` — шаг-guard: если с последнего успешного запуска ничего значимого
не изменилось, шаги после него пропускаются.

```lua
--- скомпилировать protobuf-описания
task "gen" {
    unchanged { "proto/**/*.proto", outputs = "src/generated" },
    "protoc --rust_out src/generated proto/*.proto",
}
```

Первый запуск: guard записывает фингерпринт, шаги выполняются. Следующие: если
фингерпринт совпал **и** `outputs` на месте, остаток таска пропускается. Шаги
*до* guard'а выполняются всегда.

## Что входит в фингерпринт

| Часть | Меняется, когда |
|---|---|
| входные файлы | файл по glob'ам добавлен, удалён или изменён (размер или mtime — содержимое не читается, поэтому проверка быстрая и на больших деревьях) |
| план шагов | изменились сами шаги таска |
| переменные | изменился параметр или оверрайд `--vars` |
| окружение | изменилась любая переменная окружения (вызываемая утилита может читать любую из них) |

Замечания:

- Позиционные элементы — glob'ы относительно каталога Takufile; `**`
  рекурсивен.
- `outputs` — путь или список путей. Отсутствующие outputs форсируют
  пересборку даже при совпавшем фингерпринте — удаление артефактов «просто
  работает».
- Файлы учитываются по размеру + mtime, поэтому `touch` даёт одну лишнюю
  пересборку. Ошибка в безопасную сторону.
- Переменные, выставленные function-шагом *ниже* guard'а, в ключ не входят.
- Состояние живёт в `.taku/state/<task>.bin` и записывается только после
  успеха всего таска. Упавший запуск состояние не записывает.

## Флаги

```sh
taku run gen --explain   # taku: gen: skip (unchanged)
                         # taku: gen: rebuild (input files changed)
taku run gen --force     # игнорировать состояние, выполнить всё
```

`--explain` называет изменившуюся часть; `--force` пересобирает и записывает
свежий фингерпринт.
