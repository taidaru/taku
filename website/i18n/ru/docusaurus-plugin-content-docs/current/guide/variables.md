# Переменные и плейсхолдеры

Каждая строка в шаге — шаблон. Раннер разрешает его непосредственно перед
исполнением шага.

| Синтаксис | Источник | Если нет |
|---|---|---|
| `${имя}` | переменные таска | ошибка |
| `$NAME`, `${$NAME}` | окружение, затем [`.env`](../api/env.md) | ошибка |
| `$$` | литеральный `$` | — |

«Одинокий» `$`, не подходящий ни под одну форму, — ошибка: шаблоны никогда не
ломаются молча.

**Переменные таска** приходят из трёх мест, позднее побеждает:

1. дефолты в заголовке — `<profile=dev>`;
2. командная строка — `--vars profile=release`;
3. присваивания раннего function-шага — `ctx.vars.profile = "..."`.

`--vars` принимает только имена, объявленные в заголовке. Опечатка — ошибка с
подсказкой did-you-mean.

**Переменные окружения отделены намеренно.** `$NAME` разрешается только из
окружения — переменная таска никогда его не подменит. Держите секреты в
окружении: при падении команды Taku печатает неразрешённый шаблон, так что
значения секретов не попадают в вывод и логи.

`raw("...")` оборачивает значение, которое раннер должен пропустить без
форматирования — для данных, в которых есть свой `$`:

```lua
write { raw "цена: $5", to = "note.txt" }
```

## Function-шаги

Lua-функция в списке шагов — люк для логики, которую не выразить
шагами-данными. Она получает контекст:

```lua
task "release <tag>" {
    function(ctx)
        local r = cmd.capture({ "git", "rev-parse", "--short", "HEAD" })
        ctx.vars.sha = r.stdout:gsub("%s+$", "")
    end,
    "git tag -a ${tag} -m 'релиз ${tag} (${sha})'",
}
```

- `ctx.vars` — живая таблица переменных таска. Выставленные значения питают
  плейсхолдеры всех последующих шагов.
- `ctx.task` — spec таска: имя, параметры, deps, doc.
- `fmt("...")` — форматирует шаблон текущими переменными, для использования
  внутри функции.

Внутри function-шага [модульные API](../api/index.md) — `cmd`, `fs`, `net`,
`env` — совершают эффекты напрямую.

## Загрузка и исполнение

Takufile сначала **загружается** (чтобы узнать таски), и лишь потом его таски
**исполняются**. Эффекты — команды, запись файлов, сеть — разрешены только во
время исполнения таска. Вызов `cmd.run` на верхнем уровне Takufile — ошибка;
чтения вроде `fs.read`, `fs.glob` и `env.get` работают в обеих фазах. Поэтому
`taku list` и `--dry-run` гарантированно не трогают систему.
