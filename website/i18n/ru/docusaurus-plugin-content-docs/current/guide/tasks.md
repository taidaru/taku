# Таски

Таск — это строка-заголовок плюс список шагов:

```lua
--- собрать проект
task "build <profile=dev>: gen" {
    unchanged { "src/**/*.rs", outputs = "target" },
    "cargo build --profile ${profile}",
}
```

Lua строит список шагов при загрузке Takufile; раннер исполняет его по
`taku run build`. Ничего из тела таска не выполняется на этапе загрузки.

## Заголовок

```
"имя <парам> <парам=дефолт>: dep1 dep2"
```

| Часть | Значение |
|---|---|
| `имя` | имя таска — для `taku run <имя>` и списков зависимостей |
| `<парам>` | параметр без значения по умолчанию — обязан прийти из `--vars парам=...` |
| `<парам=дефолт>` | параметр со значением по умолчанию |
| `: dep1 dep2` | зависимости, выполняются раньше таска |

Параметры становятся плейсхолдерами `${парам}` в шагах таска (см.
[Переменные](./variables.md)). `--vars` принимает только объявленные имена;
опечатка получает подсказку did-you-mean.

## Doc-комментарии

Блок строк `---` прямо над `task` документирует его. `taku list` показывает
первую строку:

```lua
--- деплой в прод
--- требует TOKEN в окружении
task "deploy <env=staging>" {
    ...
}
```

## Зависимости

`taku run <task>` сначала выполняет транзитивные зависимости. Правила:

- Каждый таск выполняется **не более одного раза за запуск**, сколько бы
  тасков от него ни зависело.
- Независимые таски идут **параллельно**, до `-j` (по умолчанию — число CPU).
- Неизвестные зависимости и циклы сообщаются до начала работы.
- Первая ошибка останавливает планирование; уже запущенные таски дорабатывают.

**Агрегатор** — таск с пустым списком шагов, именованная группа зависимостей:

```lua
task "ci: test lint build" {}
```

## Две формы вызова

Каррированная форма `task "заголовок" { шаги }` — каноническая. Форма со
скобками `task(header, steps)` делает то же и нужна, когда заголовок —
выражение:

```lua
for _, name in ipairs({ "core", "ui", "api" }) do
    task("check-" .. name, {
        "cargo check -p " .. name,
    })
end
```

Повторное определение имени печатает предупреждение; побеждает последнее.

## Разбиение Takufile

`import` загружает другой `.lua`-файл в тот же набор тасков:

```lua
-- Takufile.lua
import("tasks/build.lua")
```

- Пути относительны файлу, вызвавшему `import` (вложенные импорты — своему
  файлу).
- Файл выполняется в той же [песочнице](../sandbox.md) с теми же глобалами.
- Каждый файл импортируется не более одного раза, так что два файла могут
  разделять общий хелпер.

## Изоляция

Каждый таск исполняется в собственном свежем состоянии Lua: общего Lua-состояния
между тасками нет, а код верхнего уровня Takufile выполняется по разу на
каждый исполняемый таск. Держите на верхнем уровне только определения тасков.
