# CLI

```sh
taku init              # записать стартовый Takufile.lua
taku list              # таски со строками описания          (псевдоним: ls)
taku run <task>        # выполнить таск и его зависимости    (псевдоним: r)
```

`Takufile.lua` читается только из текущего каталога. Запуск `taku` без команды
печатает справку.

## Флаги `taku run`

| Флаг | Эффект |
|---|---|
| `-n`, `--dry-run` | напечатать дерево зависимостей и каждый шаг; ничего не исполнять |
| `--vars KEY=VAL` | задать параметр таска (повторяемый; только объявленные в заголовке имена) |
| `-y`, `--yes` | авто-ответ на шаги `confirm` |
| `-f`, `--force` | выполнить, даже если guard `unchanged` не видит изменений |
| `--explain` | напечатать, почему guard `unchanged` пропустил или пересобрал |
| `-j`, `--jobs N` | ограничить параллелизм (по умолчанию — число CPU) |

`--vars` применяется к запрошенному таску, не к его зависимостям.

## Dry run

`--dry-run` печатает дерево зависимостей, затем шаги каждого таска в порядке
исполнения:

```
dev
├─ build
└─ api

build:
  unchanged: the remaining steps would run
  cargo build --profile ${profile}
  <lua Takufile.lua:12>
api:
  serve "cargo run -p api" ready={http="http://127.0.0.1:8000/health";}
```

Шаблоны команд остаются неразрешёнными (секреты не печатаются),
function-шаги показываются как `<lua файл:строка>`, guard `unchanged`
сообщает, что пропустил бы настоящий запуск. Dry run никогда не трогает
систему.

## Код выхода

`taku run` завершается ненулевым кодом, если упал любой таск: команда вернула
ненулевой код (без `allow_fail`), отклонён `confirm`, умер сервис или брошена
Lua-ошибка. Первая ошибка останавливает планирование новых тасков.
