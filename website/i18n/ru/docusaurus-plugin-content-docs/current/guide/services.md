# Сервисы

`serve` запускает долгоживущий процесс — dev-сервер, базу данных — переживающий
свой шаг. Он обязан быть **последним** шагом таска.

```lua
--- API-сервер
task "api: migrate" {
    serve {
        "cargo run -p api",
        ready = { http = "http://127.0.0.1:8000/health", timeout = 30 },
    },
}

--- весь dev-стек
task "dev: api web" {}
```

Формы:

```lua
serve "npm run dev"                          -- просто команда
serve { "npm run dev", cwd = "frontend" }    -- с опциями (cwd, env, ready)
serve { api = { "..." }, web = { "..." } }   -- несколько сервисов в одном таске
```

## Готовность

`ready` определяет, когда таск считается выполненным и зависимые могут
продолжать:

| `ready` | Таск завершается, когда |
|---|---|
| *(отсутствует)* | сразу после старта процесса |
| `{ timeout = сек }` | через `сек` секунд |
| `{ http = "url", timeout = потолок }` | URL ответил **2xx**; `timeout` ограничивает ожидание (по умолчанию 30 с) |

Всё остальное — connection refused, 5xx от прогревающегося сервера — HTTP-проба
считает «ещё не готов» и продолжает опрос. Таск роняет только истёкший таймаут.

## Жизненный цикл

- Как **зависимость** сервис стартует в фоне; после готовности граф
  продолжается. По завершении запуска сервисы гасятся.
- Когда **всё** запрошенное — сервисы (или их агрегаторы), Taku держит их:
  `taku run dev` печатает `services running — press Ctrl+C to stop`, и Ctrl+C
  останавливает сервисы вместе с ним.

Правила выхода — сервис должен переживать запуск, поэтому ранний выход — знак
проблемы:

- вышел до готовности → его таск падает;
- вышел с ненулевым кодом в любой момент → запуск падает и **все остальные
  сервисы останавливаются**;
- вышел чисто во время удержания → убирается из удержания; когда не осталось
  никого, запуск завершается нормально.
