# Введение

Taku автоматизирует рутинные задачи разработки — сборку, тесты, деплой, линтинг и
прочее. Задачи описываются на **Lua** в файле `Takufile.lua` в корне проекта.

- **Ядро на Rust** даёт Lua безопасный кроссплатформенный API (файлы, процессы,
  сеть, SSH).
- **Lua** — пользовательский слой: из этих API вы собираете задачи.

Takufile исполняется в [песочнице](./sandbox.md): собственные `io`/`os` в Lua
отключены, поэтому обращаться к системе можно только через глобалы
`fs`/`sh`/`net`/`env`/`ssh`. Задачи объявляют зависимости, а независимые
выполняются параллельно.

```lua
task("gen", {
    desc = "сгенерировать исходники",
    run = function()
        fs.write("version.txt", "1.0.0\n")
    end,
})

task("build", {
    desc = "собрать проект",
    deps = { "gen" },
    run = function()
        if sh.run({ "cargo", "build" }) ~= 0 then
            error("сборка не удалась")
        end
    end,
})
```

```sh
taku run build     # выполнит `gen`, затем `build`
taku list          # покажет все задачи
```
