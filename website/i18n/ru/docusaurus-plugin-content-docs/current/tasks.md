# Задачи

У задачи есть имя и тело. Две формы записи:

```lua
-- простая функция: без зависимостей
task("hello", function()
    print("привет")
end)

-- таблица-спецификация
task("build", {
    desc = "собрать проект",        -- необязательно; видно в `taku list`
    deps = { "gen", "fmt" },        -- необязательно; выполняются первыми
    run = function() ... end,       -- обязательно
})
```

## Зависимости

`taku run <task>` сначала выполняет транзитивные зависимости, каждую **не более
одного раза**. Неизвестные задачи и циклы зависимостей сообщаются как ошибки до
начала выполнения.

## Параллелизм

Независимые задачи выполняются одновременно в отдельных потоках. Первая ошибка
останавливает запуск новых задач; уже запущенные завершаются.

Тело каждой задачи исполняется в **своей свежей песочнице** — общего изменяемого
Lua-состояния между задачами нет, а код верхнего уровня Takefile выполняется по
разу на каждую запускаемую задачу. Держите на верхнем уровне только определения
задач.

## Разбиение на файлы

```lua
import("tasks/build.lua")   -- путь относительно импортирующего файла
```

`import` выполняет другой `.lua`-файл в той же песочнице, не более одного раза.
Импортированный файл регистрирует свои задачи в общий набор.
