# API-модули

Модули совершают эффекты **немедленно**. Они предназначены для
`function(ctx)`-шагов — люка, — тогда как обычные тела тасков используют
[шаги](../guide/steps.md), которые раннер исполняет сам.

| Модуль | Назначение |
|---|---|
| [`cmd`](./cmd.md) | запуск команд: `run`, `try`, `capture` |
| [`fs`](./fs.md) | файловая система: чтение, запись, копирование, glob, ... |
| [`net`](./net.md) | HTTP(S)-запросы, скачивание, сырой TCP |
| [`env`](./env.md) | переменные окружения и `.env` |

Общие правила:

- Эффекты доступны **только во время исполнения таска**. Вызов на верхнем
  уровне Takufile — ошибка; чтения (`fs.read`, `fs.glob`, `env.get`, ...)
  работают и при загрузке.
- Пути — строки. Содержимое файлов и вывод команд — **байтовые строки**,
  бинарные данные проходят без искажений.
- Ошибки поднимаются как Lua-ошибки с контекстом; при необходимости ловите
  их `pcall`.
- Модули **не** применяют `${...}`-форматирование к аргументам — используйте
  `fmt("...")`, когда нужны плейсхолдеры (см.
  [Переменные](../guide/variables.md)).

Также глобальны: `task`, `import`, `fmt`, `raw` и все
[конструкторы шагов](../guide/steps.md). Из библиотек Lua загружены `string`,
`table`, `math`, `utf8` и `coroutine`; остальное отключено — см.
[Песочницу](../sandbox.md).
