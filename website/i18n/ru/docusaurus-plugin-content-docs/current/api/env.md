# env — окружение

Доступ к переменным окружения только на чтение.

```lua
local mode = env.get("MODE", "dev")   -- значение или "dev", если не задано
local token = env.require("TOKEN")    -- ошибка, если не задано
```

- `env.get(name [, default])` → значение, либо `default` / `nil`, если не задано.
- `env.require(name)` → значение, либо Lua-ошибка, если не задано.

## Автозагрузка `.env`

Если рядом с `Takufile.lua` лежит файл `.env`, Taku загружает его автоматически и
использует его значения как **фолбэк**: реальная переменная окружения всегда
имеет приоритет, а `.env` лишь подставляет то, чего в окружении ещё нет.

```bash
# .env
API_URL=https://example.com
export TOKEN="abc 123"        # необязательный префикс `export `; кавычки учитываются
SECRET='no $expansion here'   # одинарные кавычки — литерал
APP_DIR=${BASE_DIR}/app       # подстановка ${VAR}
DEBUG=1 # хвостовой комментарий после значения отбрасывается
```

Поддерживаемый синтаксис: строки `KEY=VALUE`, строки-комментарии `#` и пустые
строки, необязательный префикс `export ` и значения в одинарных или двойных
кавычках (внутри двойных кавычек раскрываются экранирования `\n`, `\\` и `\"`).
Значения могут ссылаться на другие переменные через `${VAR}`: сначала смотрится
реальное окружение, затем переменные, определённые выше в этом же файле;
значения в одинарных кавычках берутся буквально. Файл только читается (на этой
машине) и никогда не перезаписывается.

Тот же порядок поиска стоит за плейсхолдерами `$NAME` / `${$NAME}` в шаблонах
шагов, а каждый процесс, запущенный шагом, получает значения из `.env` для
переменных, которые окружение оставило незаданными.
