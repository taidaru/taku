# sh — команды

Команда — это **список аргументов**, запускаемый напрямую, без оболочки.
Поведение не зависит от `bash`/`cmd`, а подставленные значения не переразбираются
(нет инъекций).

```lua
sh.run({ "cargo", "build" })             -- стримит stdio, возвращает код выхода

local msg = 'release: "v1.0" (финал)'
sh.run({ "git", "commit", "-m", msg })   -- msg не нужно квотить и экранировать

local r = sh.capture({ "git", "rev-parse", "HEAD" })
print(r.code, r.stdout, r.stderr)
```

- `sh.run(argv [, opts])` → код выхода. Наследует stdio (вывод в реальном времени).
- `sh.capture(argv [, opts])` → `{ code, stdout, stderr }`. `stdout`/`stderr` —
  байтовые строки.

Ненулевой код выхода **возвращается**, а не выбрасывается — проверяйте `code`
сами. Ошибка Lua возникает только если команду не удалось запустить (например,
программы нет в `PATH`).

Строка вместо списка отклоняется. Если нужны возможности оболочки (пайпы, globs,
перенаправления), запустите оболочку явно:

```lua
sh.run({ "sh", "-c", "echo hello | tr a-z A-Z | tee log" })
```

## Опции

Одни и те же `opts` действуют и для `sh.run`, и для `sh.capture`:

```lua
fs.mkdir("build")
sh.run({ "sh", "-c", "echo $CC && pwd && cat" }, {
    cwd = "build",              -- рабочий каталог
    env = { CC = "clang" },     -- доп. переменные окружения (добавляются к унаследованным)
    stdin = "данные на stdin",  -- передаются команде на stdin
})
```
