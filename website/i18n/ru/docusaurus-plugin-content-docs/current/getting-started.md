# Начало работы

Установите Taku ([все способы](installation.md)) и создайте стартовый файл в
корне проекта:

```sh
taku init
```

`taku init` записывает `Takufile.lua` с примерами и комментариями.
Минимальный выглядит так:

```lua
--- прогнать тесты
task "test" {
    "cargo test",
}

--- форматирование и линт
task "lint" {
    "cargo fmt --check",
    "cargo clippy",
}

--- всё, что запускает CI
task "ci: test lint" {}
```

Три таска: `test` и `lint` выполняют команды, `ci` — агрегатор: своих шагов
нет, он только тянет зависимости (имена после `:` в заголовке).

```sh
taku list        # все таски со строкой описания
taku run ci      # test и lint, параллельно
taku run lint    # только lint
```

Полезно с первого дня:

```sh
taku run ci --dry-run    # показать, что выполнится, ничего не запуская
taku run ci -j 1         # последовательный вывод
```

Полный справочник команд и флагов — в [CLI](./guide/cli.md). Как устроены
таски, шаги и плейсхолдеры — в [Руководстве](./guide/tasks.md).

Замечания:

- `Takufile.lua` читается только из текущего каталога; родительские каталоги
  не просматриваются.
- Файл `.env` рядом с Takufile загружается автоматически — см.
  [env](./api/env.md).
